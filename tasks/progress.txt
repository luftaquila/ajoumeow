## Codebase Patterns
- Monorepo: client/ (Vue 3 + Vite) and server/ (Fastify + Drizzle ORM), root package.json uses concurrently
- TypeScript base config at root tsconfig.json, extended by client/tsconfig.app.json and server/tsconfig.json
- ESLint flat config (eslint.config.js) with typescript-eslint + eslint-plugin-vue + prettier
- Server uses ES modules ("type": "module"), tsx for dev, tsc for build
- Client uses Vite with @vitejs/plugin-vue, vue-tsc for typecheck
- Vite dev proxy: /api → http://localhost:5710 (Fastify server port)
- Old server node_modules have permission issues from Docker (sharp vendor files) - can be ignored, new deps installed alongside
- Server .env exists with JWTSecret key (will need JWT_SECRET for new stack)
- Run `npm run typecheck` from root to check both client and server
- Fastify app factory pattern: server/src/app.ts exports `buildApp()`, server/src/index.ts calls it
- @fastify/static serves gallery images from server/data/gallery/ with prefix /gallery/
- Global error handler returns `{ error: string, statusCode: number }` JSON format
- Graceful shutdown handles SIGINT/SIGTERM in server/src/index.ts
- Server .env has JWT_SECRET, PORT keys; loaded via `import 'dotenv/config'` in index.ts
- Drizzle SQLite: use `integer('col', { mode: 'boolean' })` for booleans, `sql\`(datetime('now'))\`` for timestamp defaults
- Drizzle SQLite: composite PKs use `primaryKey({ columns: [...] })` in the table's third arg callback
- Schema is at server/src/db/schema.ts; drizzle.config.ts points to ./data/ajoumeow.db
- DB connection at server/src/db/index.ts exports `db` (Drizzle instance) and `sqlite` (raw better-sqlite3)
- WAL mode is set on app startup via `sqlite.pragma('journal_mode = WAL')`
- Drizzle migrations are in server/drizzle/; run `npm run db:generate` then `npm run db:migrate` from root
- .gitignore must NOT have `*.sql` (blocks migration files); drizzle migration files + meta must be tracked
- Migration script at scripts/migrate.ts uses `mariadb` package (devDependency at root) and direct better-sqlite3/Drizzle for SQLite writes
- MariaDB connection env vars: MARIA_HOST, MARIA_PORT, MARIA_USER, MARIA_PASSWORD, MARIA_DATABASE
- Old namelist_ columns: college, department, ID (int, student_id), name, phone, birthday, 1365ID (volunteer_id), register, role
- Old register_ columns: timestamp, ID (int), name, college, department, phone
- `import * as mariadb from 'mariadb'` (no default export in the mariadb package types)
- Fastify plugins use `fastify-plugin` (fp) wrapper; plugin files go in server/src/plugins/
- Service/scheduler files go in server/src/services/; started from index.ts after app.listen()
- Request IP: use `x-forwarded-for` header (first value, split by comma) with `request.ip` fallback
- Route files go in server/src/routes/; register in app.ts via `app.register(routeModule)`
- Use Zod for request body/query validation in routes (safeParse + 400 response on failure)
- Route pattern: export default async function (app: FastifyInstance) { ... }
- Current semester lookup: `db.select().from(semesters).where(eq(semesters.isCurrent, true))`
- Use native `fetch` instead of `axios` for external HTTP calls (Node 18+ built-in, no extra dependency)
- Zod query param pattern: `.optional().default('1').transform(Number)` — `.default()` must come BEFORE `.transform()` since query params arrive as strings
- Gallery tag aggregation: LEFT JOIN photo_tags → photos with GROUP BY for per-tag photo count and likes sum
- For IN clause in Drizzle: `sql\`${col} IN (${sql.join(ids.map(id => sql\`${id}\`), sql\`, \`)})\``
- Google Apps Script macro URL stored in `MACRO_1365_URL` env var; old field names (cheif, volID) kept for backward compatibility
- Drizzle `between()` import from `drizzle-orm` for inclusive range queries on text date columns
- @fastify/multipart: `request.file()` returns file + fields; access field value via `(field as { value: string }).value`
- Gallery files saved to server/data/gallery/; thumbnails use `thumb_` prefix; filenames are timestamp-based
- Weather service: in-memory cache with fallback to web/res/weather.json; scheduler in services/, route in routes/
- Weather data format: `{ current: { weather, temp, tempSense, icon, dust: { pm10, pm25 } }, forecast: [...], update }`
- Client uses axios with baseURL '/api', token from localStorage attached via `x-access-token` header interceptor
- Pinia stores use composition API style: `defineStore('name', () => { ... })`
- Auth store: `useAuthStore()` — user, token, isLoggedIn, isAdmin, login(), autologin(), logout()
- Settings store: `useSettingsStore()` — announcement, currentSemester, fetchAnnouncement(), fetchCurrentSemester()
- Router admin guard applied on `/admin` parent route; all admin child routes are auto-protected
- Stub page components needed for vue-tsc to resolve lazy imports — each page is a minimal `<template>` placeholder until implemented
- TailwindCSS v3 installed in client/ with postcss.config.ts + tailwind.config.ts; global CSS at src/assets/main.css imported in main.ts
- DefaultLayout.vue at client/src/layouts/DefaultLayout.vue wraps all pages with header + main + footer
- App.vue mounts DefaultLayout; DefaultLayout contains RouterView for page content
- Vite dev proxy: /api AND /gallery both proxied to http://localhost:5710
- Primary color palette: orange-based (primary-500: #ee7711) — matches cat club theme
